🔒 安全审计 Security Audit
自动记录所有 Agent 工具调用日志，包含时间、工具名称、决策结果及原因。 Logs every agent tool call with timestamp, tool name, decision and reason.
功能说明
拦截所有智能体的工具调用请求，将调用记录持久化到 ~/.openharness/audit/tool_calls.jsonl。
  • 记录内容包括：timestamp、tool_name、arguments、decision（allowed/denied/pending）、reason、agent_id
  • 支持事后回溯：查看谁在什么时间调用了什么工具，结果如何
  • 异常行为检测：短时间内高频调用敏感工具时会自动标记
 How it works
Intercepts every tool call from any agent and persists audit records to ~/.openharness/audit/tool_calls.jsonl.
  • Logs: timestamp, tool_name, arguments, decision (allowed/denied/pending), reason, agent_id
  • Post-hoc trace: see who called what tool, when, and the result
  • Anomaly detection: flags high-frequency sensitive tool calls
✅ 已集成到系统Built-in v0.6.78
⛔ 熔断器 Circuit Breaker
API 连续失败后自动跳闸，快速拒绝而非等待超时，后台自动探测恢复。 Trips after consecutive API failures — fail-fast instead of waiting for timeout.
功能说明
当 API 连续失败达到阈值后自动"跳闸"，后续请求不再发送，立即返回熔断状态。
  • 触发条件：连续 5 次 API 调用失败（网络错误、5xx、限流）
  • 跳闸效果：后续请求在毫秒内返回"暂时不可用"，不等超时
  • 自动恢复：30 秒后发一次探测请求，成功后自动恢复
  • 覆盖范围：Anthropic Claude + 所有 OpenAI 兼容协议（DeepSeek、Kimi、DashScope 等）
 How it works
Automatically trips when consecutive API failures reach the threshold.
  • Threshold: 5 consecutive failures (network errors, 5xx, rate limits)
  • Effect: Subsequent requests fail instantly instead of timing out
  • Recovery: Auto-probes after 30s, resumes when healthy
  • Coverage: Anthropic Claude + all OpenAI-compatible APIs (DeepSeek, Kimi, DashScope, etc.)
✅ 已集成到系统Built-in v0.6.78
🔄 工具错误重试 Tool Error Retry
工具调用因临时网络错误失败时自动重试一次，避免偶发抖动导致任务中断。 Auto-retries on transient network errors to prevent task interruption.
功能说明
工具执行层内嵌重试逻辑，仅对临时性错误触发。
  • 触发条件：DNS 解析失败、连接重置、TLS 握手失败、瞬态 5xx
  • 重试次数：最多 1 次（网络抖动往往一次就够）
  • 安全机制：不会重试已执行的非幂等操作（如文件创建、数据库写入）
  • 错误分类：内部有 is_retryable() 函数精确区分可重试与不可重试错误类型
 How it works
Built-in retry logic at the tool execution layer.
  • Triggers on: DNS failures, connection resets, TLS handshake errors, transient 5xx
  • Max retries: 1 (single retry is usually enough for transient issues)
  • Safety: Never retries non-idempotent operations (file creation, DB writes)
  • Classification: Internal is_retryable() function distinguishes retryable vs non-retryable errors
